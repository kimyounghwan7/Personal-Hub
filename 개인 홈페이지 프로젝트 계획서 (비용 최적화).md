# **개인 홈페이지 및 생산성 허브 프로젝트 계획서 (비용 최적화 버전)**

본 문서는 Next.js, FastAPI, Supabase를 활용한 개인 홈페이지 및 생산성 관리 서비스 구축 계획서입니다. 월 5만 원 이하(최대 무료)의 예산으로 운영하기 위해 클라우드 네이티브 무료/저비용 플랫폼을 활용하는 아키텍처로 설계되었습니다.

## **1\. 프로젝트 개요**

* **프로젝트 명:** Personal Hub (가칭)  
* **주요 목적:** 개인 포트폴리오 전시 및 실무적인 생산성 도구 통합  
* **목표 예산:** 월 0원 \~ 10,000원 (트래픽에 따라 변동 가능)

## **2\. 비용 최적화 기술 스택 (Tech Stack)**

| 구분 | 기술 및 플랫폼 | 비용 (예상)   |
| :---- | :---- | :---- |
| **Frontend** | Next.js 배포: **Vercel** | Hobby Tier (무료) |
| **Backend** | FastAPI 배포: **Render** 또는 **Fly.io** | Free Tier \~ 월 $5-$7 |
| **Database/Auth** | **Supabase** | Free Tier (무료) |
| **Domain** | 개인 도메인 구매 (선택) | 연 1\~2만 원 |

## **3\. 주요 기능 명세**

### **3.1. 자기소개 (About Me)**

* 소프트웨어 엔지니어 및 AI 연구 역량 전시.  
* 진행 프로젝트 및 기술 스택, GitHub 링크 연결.

### **3.2. To-Do 리스트**

* Supabase 연동 CRUD 및 우선순위, 기한(Due Date) 관리.

### **3.3. 캘린더 연동**

* To-Do의 기한 데이터를 달력 UI에 매핑.

## **4\. 데이터베이스 설계 (Supabase)**

* **Profiles:** 사용자 기본 정보.  
* **Todos:** id, user\_id, title, description, status, due\_date, priority

## **5\. GitHub 기반 자동 배포 전략 (CI/CD)**

기존 AWS의 복잡한 파이프라인 대신, GitHub 연동을 통한 Zero-Config 배포를 구성합니다.

### **5.1. Frontend (Vercel)**

* Vercel 대시보드에서 GitHub 레포지토리 연동.  
* main 브랜치에 코드가 푸시되면 자동으로 Next.js 빌드 및 글로벌 CDN 배포 수행.

### **5.2. Backend (Render / Fly.io)**

* FastAPI 프로젝트 내에 Dockerfile 작성.  
* GitHub 레포지토리와 연동하여 커밋 시 Docker 컨테이너를 자동 빌드 및 서비스 업데이트.

## **6\. 개발 및 배포 마일스톤**

1. **로컬 세팅:** docker-compose.yml로 프론트/백/Supabase(로컬) 환경 통일.  
2. **기능 개발:** Next.js 컴포넌트 작성 및 FastAPI 연동.  
3. **클라우드 연동:** Supabase 클라우드 프로젝트 생성 및 테이블/Auth 룰(RLS) 세팅.  
4. **자동 배포:** Vercel과 Render에 GitHub 레포 연결 및 라이브 서버 테스트.
